axios、jquery的ajax用法

04 / 26

前两天爬虫实验中，看到了HTTP请求中Post请求发送的是Request Payload形式与传统fromData不同。特此记录.

HTTP请求中 request payload 和 formData 区别？

FromData和Payload是浏览器传输给接口的两种格式。 这两种方式浏览器是通过Content-Type来进行区分的 。如果是 application/x-www-form-urlencoded的话，则为formdata方式，如果是application/json或multipart/form-data的话，则为 request payload

Ajax

实验环境

后端：Django

前端：Axios、JQuery-Ajax

  • JQuery 3.5.0
  • Axios 0.19.2

用Ajax的方式提交Post代码

<script>
    var data = {
        'name': 'tao',
        'age': 21
    };
    $('.send').click(()=>{
        $.ajax({
            url: 'http://127.0.0.1:8000/answer',
            type: 'POST',
            dataType: 'json',
            data: data,
            success: (res)=>{
                console.log(res)
            }
        })
    })
</script>

那么在后端的响应的数据是这样子的

def answer(request):
     # 如果请求头中的: Content-Type: application/x-www-form-urlencoded   request.POST中才会有值（才会去request.body中解析数据）
    print(request.body)
    print(request.POST)
    print(request.POST.get('name'))

    return HttpResponse('ok')

使用Content-Type: application/json 编码

<script>
    var data = {
        'name': 'tao',
        'age': 21
    };
    $('.send').click(()=>{
        $.ajax({
            url: 'http://127.0.0.1:8000/answer',
            type: 'POST',
            dataType: 'json',
            contentType: 'application/json',  // *
            data: JSON.stringify(data),  // *
            success: (res)=>{
                console.log(res)
            }
        })
    })
</script>


b'{"name":"tao","age":21}'
<QueryDict: {}>
None

这时候可以看到QueryDict捕捉不到数据了。而request.body依旧存在数据. 解决办法是从 request.body里获取数据 然后再通过json.loads(request.body)

def answer(request):
    print(request.body) 
    print(request.POST)
    print(request.POST.get('name'))

    data = json.loads(request.body)
    print(data, type(data))
    return HttpResponse('ok')
>>>
b'{"name":"tao","age":21}'
<QueryDict: {}>
None
{'name': 'tao', 'age': 21} <class 'dict'> # 可以得到数据

Axios使用

这是一个最简单的POST请求

<script>
    var data = {
        'name': 'tao',
        'age': 21
    };
    $('.send').click(()=>{
        axios.post('/answer', data=data)
            .then((res)=>{
                console.log(res);
            })
    })
</script>

可以看到默认的就是request payload . Axios也将默认的请求类型设置为了application/json;

Request Payload请求

  • 请求头部的 Content-Type: application/json
  • 请求正文是一个 json 格式的字符串
{"name":"tao","age":21}

FromData请求

  • 请求头部的 Content-Type: application/x-www-form-urlencoded
  • 请求正文是类似 Get 请求 url 的请求参数
name=tao&age=21

总结

  1. request payload form data 关键在于请求头中的Content-type

  2. 不同的格式在后台进行处理的结果是不一样的。

  3. 在进行爬虫爬取的时候如果是RequestPayload的话需要将请求数据转换为Json格式字符串 就ok了。
