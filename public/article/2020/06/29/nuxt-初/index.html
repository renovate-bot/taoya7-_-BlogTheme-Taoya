Nuxt.js-初

06 / 29

Nuxt

Github

是什么？

Nuxt.js 是一个基于 Vue.js 的通用应用框架。

特点

  1. 基于Vue.js
  2. 服务端渲染
  3. 强大的路由功能
  4. 静态文件服务

服务端渲染SSR

单页应用程序SPA

首次使用

创建项目

$ npx create-nuxt-app <项目名>

//Or

$ yarn create-nuxt-app <项目名>

// Or
$ vue init nuxt-community/starter-template <项目名称>



# 目录结构
.nuxt: 
assets: 资源文件。
components: 组件，存放页面可复用组件。
layouts: 布局，默认default。
middleware: 中间件.权限|认证
pages: 页面目录,一个Vue文件即一个页面。
plugin: 插件目录。
static: 静态文件。
store: 状态管理。
nuxt.config.js: 配置文件。

目录

pages目录

Nuxt.js 会依据 pages 目录中的所有 *.vue 文件生成应用的路由配置。

默认是pages/index.vue

在pages/新建一个about.vue

然后打开127.0.0.1:3000/about

路由

Nuxt.js 依据 pages 目录结构自动生成 vue-router 模块的路由配置。（无需手动配置）

基础路由

路由跳转

要在页面之间使用路由，使用<nuxt-link>标签

<nuxt-link to='/about'>关于</nuxt-link>

路由映射

因为不需要配置路由，所以要知道映射规则

例子：

pages/
--| user/
-----| index.vue
-----| setting.vue
--| index.vue

router: {
  routes: [
    {
      name: 'index',
      path: '/',
      component: 'pages/index.vue' // 默认
    },
    {
      name: 'user',
      path: '/user',
      component: 'pages/user/index.vue'
    },
    {
      name: 'user-setting',
      path: '/user/setting',
      component: 'pages/user/setting.vue'
    }
  ]
}

/
/user
/user/setting

动态路由

在 Nuxt.js 里面定义带参数的动态路由，需要创建对应的以下划线作为前缀的 Vue 文件 或 目录。

比如创建一个pages/user/_id.vue

// 路由映射
{
    name: 'user-id',
    path: '/user/:id?',
    component: 'pages/user/_id.vue'
}

<template>
    <div>
        用户 {{$route.params}}
    </div>
</template>

访问http://127.0.0.1:3000/user/1/

路由参数校验

Nuxt.js 可以让你在动态路由组件中定义参数校验方法。

<script>
export default {
    validate({ params }){
        console.log(params);
        return /^\d+$/.test(params.id)
    }
}
</script>

嵌套路由

需要添加一个 Vue 文件，同时添加一个与该文件同名的目录用来存放子视图组件。

pages/
--| users/
-----| _id.vue
-----| index.vue
--| users.vue

router: {
  routes: [
    {
      path: '/users',
      component: 'pages/users.vue',
      children: [
        {
          path: '',
          component: 'pages/users/index.vue',
          name: 'users'
        },
        {
          path: ':id',
          component: 'pages/users/_id.vue',
          name: 'users-id'
        }
      ]
    }
  ]
}
