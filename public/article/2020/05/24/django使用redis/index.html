使用Redis进行缓存

05 / 24

减轻服务器压力的方法之一是缓存数据。这是通过在处理完数据后缓存数据，然后在下次请求时从缓存中提供数据来完成的。

准备

  • Django
  • django-redis
  • redis
pip install django
pip install django-redis
pip install djangorestframework

要在我们的应用程序中使用Redis，我们需要执行以下操作

  1. 检查缓存中是否存在当前查询的结果
  2. 如果结果存在于缓存中。就检索
  3. 如果结果不存在将其获取。将其存储在缓存中。然后转发请求。

Python应用配置Redis

为了在Django应用程序中使用Redis，我们需要设置Redis.然后将以下内容添加到您的settings.py文件中：

CACHES = {
    'default': {
        'BACKEND': 'django_redis.cache.RedisCache',
        'LOCATION': 'redis://127.0.0.1:6379/',
        'OPTIONS': {
            'CLIENT_CLASS': 'django_redis.client.DefaultClient',
        }
    }
}

from rest_framework.decorators import api_view
from rest_framework.response import Response

from .serialiers import ProductSerializer
from .models import Product


@api_view(['GET'])
def view_goods(request):
    goods = Product.objects.all()
    result = ProductSerializer(goods, many=True).data
    return Response(result)

缓存

创建视图

from django.conf import settings
from django.core.cache.backends.base import DEFAULT_TIMEOUT
from django.core.cache import cache


CACHE_TTL = getattr(settings, 'CACHE_TTL', DEFAULT_TIMEOUT)

@api_view(['GET'])
def view_goods_cached(request):
    if 'product' in cache:
        products = cache.get('product')
        return Response(products, status=200)
    else:
        products = Product.objects.all()
        result = ProductSerializer(products, many=True).data
        cache.set('product', result, CACHE_TTL)
        return Response(result)

更新url.py

from django.urls import path

from .views import view_goods, view_goods_cached
urlpatterns = [

    path('goods/', view_goods),
    path('goods_cache/', view_goods_cached)

总结

体验了Redis给应用程序带来了速度的错觉。利用Redis来查询数据结果。然后再后续的查询中直接从缓存中返回这些结果。而不是每次都返回数据库的信息。
